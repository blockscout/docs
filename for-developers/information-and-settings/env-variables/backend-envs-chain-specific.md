# Backend ENVs: Chain-Specific

## <mark style="background-color:orange;">Chain Specific ENVs</mark>

The following ENVs are specific to different types of chains and rollups. To use these specific variables you will also need to set the CHAIN\_TYPE variable to the appropriate chain.

| Variable    | Description                                                                                                                                                                                                                                     | Parameters                                                                                                    |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| CHAIN\_TYPE | Specifies the model of data and enables fetchers specific to the chain. Available values are `ethereum`, `optimism`, `arbitrum`, `polygon_zkevm`, `polygon_edge`, `rsk`, `shibarium`, `stability`, `suave`, `zetachain`, `filecoin`, `default`. | <p>Needs Recompile: ☑️<br>Version: v5.3.0+<br>Default: <code>default</code><br>Applications: API, Indexer</p> |

## Time format

Can be set in format `1h` for 1 hour, `1m` for 1 minute, `1s` or `1` for 1 second, `1ms` for 1 millisecond

{% hint style="warning" %}
_**Note**_**: Before release 5.1.2, all environment variables of time format supported only integers in seconds (without dimensions) as values.**
{% endhint %}
