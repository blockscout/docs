## Token metadata is added from 2 primary sources

1. **Automated aggregators.** Blockscout queries CoinGecko and CoinMarketCap for exchange rate and market data. Projects submit an update request directly to CoinGecko, not to Blockscout. CoinGecko requires a Public Verification Post from an account tied to the project's official channels before reviewing listing or update requests. Turnaround is a few business days, or 24 hours with CoinGecko's paid Fast Pass. Blockscout displays the data that whCoinGecko returns.
2. **Manual submission**. Requires a Blockscout account and proof of token ownership. To submit, sign into your account and sign a message to prove ownership, then submit token details. Initial metadata is reviewed by Blockscout admins and added once it is checked. This info is not verified on an ongoing basis, however, so a website could change content later or a verified address could update information.

## Does Blockscout require contract verification before updating token info?

Blockscout requires the token's contract source code to be verified before it will accept owner-submitted updates to name, logo, website, or socials. [Learn more about contract verification here.](/devs/verification) This ensures the code shown publicly matches what's actually deployed, and that only the real contract owner can update the listing.

Blockscout follows the same logic through its **Verified Addresses** feature:

1. Verify the contract's source code.
2. Sign in to My Account and prove ownership of the address, either by signing a message from the contract's owner wallet or via multisig signer verification.
3. Once ownership is verified, the address is listed under Verified Addresses in the account, and the owner can update token info directly without re-verifying each time.

The core difference from Etherscan: Blockscout's process runs per-instance (each Blockscout explorer instance handles its own verified addresses), and public tag requests are reviewed manually by the instance's team rather than through a centralized submissions queue.

## What should a project do if their token info is wrong on Blockscout ?

- **If the token is tracked on CoinGecko: submit the correction directly to CoinGecko.** Blockscout will reflect it automatically once CoinGecko's data updates.
- If the token isn't on CoinGecko, or the fix is logo/description/links: Update through the Blockscout interface.
- To update on Blockscout:

1. Log in to your account (or create an account if you don't yet have one). Go to the token page, click the 3 dots and and select **Update Token Info.**
