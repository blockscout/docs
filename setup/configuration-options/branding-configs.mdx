<Info>
Note: if you've previously deployed and are updating your BlockScout version with new assets, be sure to remove static assets from the previous build (use `mix phx.digest.clean`or manually delete assets located in *apps/block\_scout\_web/priv/static* folder) before updating current files and restarting/rebuilding BlockScout.

In order to rebuild new front-end assets run the following. *Note: If preferred, use *`npm ci`* rather than *`npm install`* to strictly follow all package versions in *`package-lock.json`

1. `cd apps/block_scout_web/assets; npm install && node_modules/webpack/bin/webpack.js --mode production; cd -`

2. `mix phx.digest`

</Info>

1. **Theme Colors and other CSS-based attributes**: See [CSS Configs and Presets](/setup/configuration-options/css-configuration-and-presets) to set your instance to the stylesheet you want to use.
2. **Logos**: Use the LOGO and FOOTER\_LOGO [env variables](/setup/env-variables) to direct to your uploaded logos. Logos and other assets are located in the<br/>`apps/block_scout_web/assets/static/images` folder.
3. **Coin**: [Customize the coin symbol](/faqs/faqs#how-do-i-customize-the-coin-symbol-name) using the COIN & COINGECKO\_COIN\_ID [env variables](/setup/env-variables).
4. **MetaData**: MetaTags can be adjusted for various pages in the templates area. For the home page, tag data is rendered from here:<br/>`/apps/block_scout_web/lib/block_scout_web/templates/chain/_metatags.html.eex`
5. **Titles / Subtitles**: Browser tab displays the title from the 2 SUBNETWORK + NETWORK [env variables](/setup/env-variables).
6. **Favicon**: replace the current favicons located in the<br/>`apps/block_scout_web/assets/static/images` folder.
7. **Menus**: Use the APPS\_MENU [env variable](/setup/env-variables) to include an apps menu and EXTERNAL\_APPS to populate the menu.
8. **Top navigation bar**: Data is rendered from here:<br/>`/apps/block_scout_web/lib/block_scout_web/templates/layout/_topnav.html.eex`
9. **Footer**: Data is rendered from here:<br/>`/apps/block_scout_web/lib/block_scout_web/templates/layout/_footer.html.eex`<br/>**ENV variables for footer links**

   a. `FOOTER_CHAT_LINK `

   b. `FOOTER_FORUM_LINK`

   c. `FOOTER_GITHUB_LINK`
   
10. **Custom Theming**: Custom theming is available from the BlockScout team. [Contact us in Discord](https://discord.com/invite/blockscout)for more information.
